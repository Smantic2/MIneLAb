# Export/Import Formats — Technical Specification

---

## 1. Format Overview

| Format | Extension | Read | Write | Edition | Tool |
|--------|-----------|------|-------|---------|------|
| Sponge Schematic | `.schem` | Yes | Yes | Java | WorldEdit, FAWE |
| Litematica | `.litematic` | Yes | Yes | Java | Litematica mod |
| Structure Block | `.nbt` | Yes | Yes | Java | Vanilla |
| Legacy Schematic | `.schematic` | Yes | No | Java | MCEdit |
| Axiom Blueprint | `.bp` | Yes | Yes | Java | Axiom |
| Bedrock Structure | `.mcstructure` | Yes | Yes | Bedrock | Structure Blocks |
| MCFunction | `.mcfunction` | No | Yes | Java | Data Packs |
| Datapack | `.zip` | No | Yes | Java | Data Packs |
| GLB/GLTF | `.glb` | Yes | Yes | N/A | Blender, 3D tools |
| OBJ | `.obj` | No | Yes | N/A | Blender, 3D tools |
| PNG | `.png` | No | Yes | N/A | Layer guides |
| MCAI (native) | `.mcai` | Yes | Yes | N/A | MIneLAb |

## 2. Sponge Schematic (`.schem`) — v2

The modern standard for WorldEdit. Based on NBT.

### Structure

```
Schematic (Root Compound)
├── Version: Int (2)
├── DataVersion: Int
├── Width: Short         (X size)
├── Height: Short        (Y size)
├── Length: Short        (Z size)
├── Offset: Int Array    [x, y, z]
├── Palette: Compound
│   ├── "minecraft:stone_bricks": Int (palette index)
│   ├── "minecraft:oak_planks": Int
│   └── ...
├── PaletteMax: Int
├── BlockData: ByteArray (palette indices, XZY order)
├── BiomePalette: Compound (optional)
├── BiomeData: ByteArray (optional)
├── Entities: List (optional)
├── BlockEntities: List (optional)
└── Metadata: Compound (optional)
```

### Block Data Encoding

Block data is a flat array of palette indices, ordered X then Z then Y (inner to outer).

```
for y in 0..height:
  for z in 0..length:
    for x in 0..width:
      index = palette[getBlockId(x, y, z)]
```

## 3. Litematica (`.litematic`)

Also NBT-based, GZIP-compressed. More complex than Sponge.

### Structure

```
Schematic (Root Compound)
├── SchematicName: String
├── Author: String
├── Description: String
├── RegionCount: Int
├── TimeCreated: Long
├── TimeModified: Long
├── MCDataVersion: Int
├── Regions: Compound
│   └── "RegionName": Compound
│       ├── Version: Int (2)
│       ├── SubRegions: Compound
│       ├── Position: Int Array [x, y, z]
│       ├── Size: Int Array [x, y, z]
│       ├── PendingBlockTicks: List
│       ├── PendingFluidTicks: List
│       ├── BlockStates: Compound
│       │   ├── Palette: List
│       │   │   └── Compound
│       │   │       ├── Name: String
│       │   │       └── Properties: Compound
│       │   └── BlockStates: Long Array
│       └── TileEntities: List
└── Metadata: Compound
```

## 4. Structure Block (`.nbt`)

Vanilla Minecraft format. Stored as NBT.

### Structure

```
Schematic (Root Compound)
├── DataVersion: Int
├── size: Int Array [x, y, z]
├── blocks: List
│   └── Compound
│       ├── state: Int
│       ├── pos: Int Array [x, y, z]
│       └── nbt: Compound (optional, for block entities)
├── palette: List
│   └── Compound
│       ├── Name: String
│       └── Properties: Compound
└── entities: List (optional)
```

---

## 5. Bedrock Structure (`.mcstructure`)

Bedrock Edition format. Also NBT-based but with different schema.

### Structure

```
Schematic (Root Compound)
├── format_version: Int (1)
├── size: Int Array [x, y, z]
├── world_origin: Int Array [x, y, z]
├── blocks: List
│   └── Compound
│       ├── name: String
│       ├── states: Compound
│       └── version: Int
├── block_indices: List
│   └── Int Array (3D flattened, block_indices[palette_index])
├── palette: List
│   └── Compound
│       └── block_palette: List
│           └── Compound
│               ├── name: String
│               ├── states: Compound
│               └── version: Int
└── entities: List (optional)
```

## 6. MCFunction / Datapack

### MCFunction

Each line is a `/setblock` or `/fill` command:

```mcfunction
# Generated by MIneLAb
# Structure: Medieval Castle (60x40x60)

# Foundation
fill 0 0 0 59 0 59 minecraft:stone_bricks
fill 1 1 1 58 1 58 minecraft:cobblestone

# Walls - North
fill 0 1 0 59 15 0 minecraft:stone_bricks
fill 5 1 0 10 8 0 minecraft:oak_stairs[facing=south]

# ... etc
```

### Datapack (`.zip`)

```
my-structure-datapack/
├── pack.mcmeta
│   └── { "pack": { "pack_format": 48, "description": "..." } }
└── data/
    └── minecraft/
        └── structures/
            └── my_structure.nbt
```

## 7. Import Priority

When importing, auto-detect format by:

1. File extension (`.schem`, `.litematic`, `.nbt`, etc.)
2. Magic bytes (NBT files start with tag type byte)
3. Content inspection (check for known NBT keys)

### Import Pipeline

```
File → Detect Format → Decompress (if GZIP) → Parse NBT → Extract Blocks → Normalize to StructureData
```

### Normalization

All formats are normalized to StructureData:

```
Format-specific blocks → Vec3 + Block → StructureData
Palette indices → Block ID mapping
Coordinate systems → Unified (X right, Y up, Z forward)
```

---

## 8. Export Pipeline

```
StructureData → Format-specific transform → Serialize → Compress (if needed) → Blob
```

### Coordinate Transform

All exports must handle:

- **Origin offset**: Structures may start at non-zero coordinates
- **Size limits**: Structure blocks max 64x64x64, Litematica has no limit
- **Block state format**: Different formats use different state representations
- **Entity data**: Some formats support entities, others don't
- **Biome data**: Optional in most formats

## 9. GLB/GLTF Export

For 3D visualization and Blender import:

```typescript
interface GLBExport {
  nodes: Array<{
    name: string;
    mesh: number;
    position: [number, number, number];
  }>;
  meshes: Array<{
    primitives: Array<{
      attributes: { POSITION: number; NORMAL: number; TEXCOORD_0: number };
      indices: number;
      material: number;
    }>;
  }>;
  materials: Array<{
    name: string;
    pbrMetallicRoughness: {
      baseColorTexture: { index: number };
      metallicFactor: 0;
      roughnessFactor: 1;
    };
  }>;
  textures: Array<{
    source: number;
  }>;
  images: Array<{
    mimeType: string;
    bufferView: number;
  }>;
}
```

Export uses greedy meshing to generate optimized geometry, then converts to glTF format with a texture atlas.

## 10. PNG Layer Export

Generates layer-by-layer images for build guides:

```
Each layer = 1 Y-level
Image dimensions = structure width x depth
Pixel color = block type color
Legend included as overlay or separate image
```

Options:
- Single image per layer
- All layers in sprite sheet
- Annotated with block counts per layer
- Numbered steps for build order
